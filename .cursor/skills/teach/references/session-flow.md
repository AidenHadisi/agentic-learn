# Session flow

The opening and closing rituals for teach-mode sessions. Use these shapes verbatim — consistency across sessions is the point.

## Opening ritual

After loading state (syllabus, progress, latest notes), open with exactly this structure:

```
**Where we are:** <topic>, unit NN "<unit name>" — <one sentence on what was covered last time and what's planned today>.

**Warm-up** (from last session):
1. <retrieval question>
2. <retrieval question>
3. <due flashcard or misconception re-test, if any>
```

Then stop and wait for answers. Grade the warm-up briefly (one line per answer) before starting new material.

If this is the first session of a topic, skip the warm-up and open with: "**Where we are:** starting <topic>, unit 01 '<unit name>'. Today: <plan>."

## During the session

- One idea per message, ending in one check-question. Wait.
- If the student goes on a tangent worth pursuing, pursue it — then log it under "Open threads" in the notes file so the syllabus thread isn't lost.
- Watch the clock against the session budget in `syllabus.md`. Better to finish one chunk well than rush three.

## Closing ritual

End every session with exactly this structure:

```
**Today:** <2-3 bullet summary of what was covered>

**You should now be able to:** <the unit objectives touched today>

**Written:** <list of files created/updated this session>

**Next time:** <what the next session starts with — be specific enough that a cold agent can run it>
```

The "Written" list is a self-check: if it's empty or missing `progress.md`, you skipped the state-writing step — go back and do it before closing.

## Interrupted sessions

If the student ends abruptly ("gotta go"), skip the summary but **never skip writing state**. Update `progress.md` with a one-line "session cut short at <point>" note so the next session resumes correctly.
